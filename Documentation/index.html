      Multi-Tenant Flat & Bill Management System

      The Flat & Bill Management System is a multi-tenant web application designed for property owners and administrators to manage flats, tenants, and billing efficiently. It provides a clear separation of roles between Admin and House Owner (User) with automated workflows such as tenant assignment, bill management, notifications, and record clean-up upon deletions.


      Core Attributes:
      • Admin :
        Create and manage house owners (users).
        Create and manage tenants.
        Assign tenants to house owners based on flat availability.
        Delete tenants (with auto-cleanup from assigned flats).
      • House Owner (User) :
        Create and manage flats.
        Assign admin-provided tenants to flats (one tenant per flat).
        Create bill categories (e.g., gas, utility, electricity).
        Create bills under flats (month-wise, amount, notes).
        Update bill status: Due → Paid.
        Delete flats (Tenant will be free to reassign by admin to houser onwer and relevant bills will also be removed).
      • Tenant :
        Assigned to only one flat at a time.
        Receives email notifications when a new bill is created, a bill is marked as paid.
      Admin Worlflow:
      • Create User (House Owner) :
        Inputs: username, email, name, country, mobile no, password.
        System sends an email with credentials to the user.
      • Create Tenant :
        Tenant is stored in DB with status = unassigned.
      • Assign Tenant to House Owner :
        Validation: House owner must have at least one available flat.
        Tenant status changes to assigned.
      • Delete Tenant :
        Tenant record removed from tenants table.
        If tenant was assigned, flat’s tenant reference is cleared.
        Tenant becomes free/unassigned in system.
      House Owner Workflow:
      • Create Flat :
        Inputs: flat number, details.
        Increases owner’s total flat count.
      • Assign Tenant to Flat :
        Only admin-provided tenants can be linked.
        One flat = one tenant.
      • Create Bill Category :
        Examples: Gas Bill, Utility Bill, Electricity Bill.
      • Create Bill :
        Inputs: category, month, amount, notes (optional).
        Bill status = Due.
        System notifies tenant via email.
      • Mark Bill as Paid :
        Updates bill status to Paid.
        System sends notification email to tenant.
        If multiple unpaid bills exist → total due is calculated under that flat.
      • Delete Flat :
        Decreases owner’s flat count.
        If a tenant was assigned → tenant becomes free/unassigned (available for reassignment).
        All bills under the flat are deleted.
      Notifications Sending:
      • User Creation: Email sent with login credentials.
        Bill Creation: Tenant receives bill details via email.
        Bill Paid: Tenant notified that payment has been marked.
